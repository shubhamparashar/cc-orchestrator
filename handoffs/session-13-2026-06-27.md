## Security audit ‚Äî COMPLETE, no exploitable vulnerabilities
Read by hand all ~16 files that touch untrusted input (the auto-mode classifier was
out, so parallel audit agents weren't available). The codebase is well-hardened; the
only real improvement found was a secret-redaction gap, shipped as v1.3.6. Surfaces
checked and found clean:
- **OS-spawn** (`actions.mjs`, `notify.mjs`) ‚Äî argv-not-shell, `osascript`/`notify-send`
  via argv items, `shq()` escaping, UUID-validated sessionId.
- **HTTP / auth / traversal** (`server.mjs`, `auth.mjs`, `contextStore.mjs`) ‚Äî
  `hostAllowed` enforced first (DNS-rebinding), no CORS headers, `X-CC` CSRF on all
  mutations, anchored `isSessionUuid` gates `/api/context|transcript/<id>` before any
  file read, static serving is a fixed allowlist, 1 MB body cap, no reflected XSS.
- **Prototype pollution** (`cost.mjs`, `config.mjs`, `pricing.mjs`) ‚Äî `isUnsafeModelKey`
  guards model keys; date keys normalized via `utcDay()`; config/pricing use object
  spread + `isRate` filter; accumulators are `Map`/`Set`.
- **ReDoS** (`scan.mjs`, `rank.mjs`, `health.mjs`, `sessionIndex.mjs`) ‚Äî transcript reads
  bounded (256 KB tail) or regexes trivially linear.
- **Secret redaction** (`history.mjs`, `diag.mjs`) ‚Äî `sanitizedEnv()` minimal, errors
  scrubbed + loopback-gated; redaction net broadened in v1.3.6.

Three candidate nits were considered and **dropped** as non-issues (argv-`--`, health
`byTool` key, `sessionTasks` guard) ‚Äî no privilege escalation / provably benign /
conflicts with hermetic fixtures.

## Releases shipped
- **v1.3.6** (PR #22) ‚Äî broader secret redaction in `lib/history.mjs` `redact()`: now
  also catches Slack incoming-webhook URLs, Google `ya29.` OAuth tokens, and bare `npm_`
  tokens (common pasted credentials the prior shapes missed). The palette is reachable
  from a phone over Tailscale and the issue URL is meant to be shared, so this matters.
- **v1.3.7** (PR #23) ‚Äî per-port session index (`indexFileForPort()` in
  `lib/contextStore.mjs`): `~/.claude/contexts/index.json` was a single shared file, so a
  second server on another port (a throwaway verify/dev instance) clobbered the primary's
  index on its reindex tick. The default port keeps the legacy `index.json` name (no
  migration); every other port writes `index-<port>.json`. Canonicalized with `Number()`
  to match the server's bind logic.
- **v1.3.8** (PR #24) ‚Äî sub-agent dialogue dedup (`dialogueBag` + shared `termBag` in
  `lib/rank.mjs`): each sub-agent search doc stored a raw ~4 KB dialogue tail; it's now a
  deduped significant-term bag over the agent's whole dialogue. Index ~32% smaller
  (3.57 ‚Üí 2.45 MB live), sub-agent bodies ~50% smaller, and recall **improves** (a term
  mentioned early in a long agent is searchable instead of lost with the old tail).
