## What this session did

### 1. Fixed a live `:7433` "Cost over time" crash (shipped in v1.1.0)
The user hit `could not load: Cannot read properties of undefined (reading 'length')` opening the cost
modal. Two causes: (a) the always-on LaunchAgent (`com.cc-orchestrator`) was a ~4-day-old process that
served the *current* `public/index.html` from disk (so the button showed) but its in-memory routes
predated `/api/cost/rollup` (+`/api/history`, `/api/diag`) → 404; (b) `loadRollup()` guarded
`r.buckets || []` into a local but then read `r.buckets.length` unguarded, so the 404 envelope threw.
Fix: hardened `loadRollup` (added the file's standard `if (!res.ok) throw` + render from the guarded
local) and **restarted the LaunchAgent** (`launchctl kickstart -k gui/$(id -u)/com.cc-orchestrator`),
which loaded the current routes. Verified live + via Claude Preview. (This fix rode into **v1.1.0**.)

### 2. Cut v1.2.0 — AFK alerts (PRODUCT_IDEAS.md §B-P3)
Opt-in OS notifications so an unattended fleet can reach you. **Default OFF** — no behavior change
until a `config.json` enables it. New files (all disjoint): `lib/config.mjs` (loader + `ALERT_DEFAULTS`,
`CC_CONFIG_DIR` override), `lib/notify.mjs` (injection-safe `osascript`/`notify-send`, headless no-op,
`CC_ALERT_DRYRUN=1` logs instead of firing), `lib/presence.mjs` (best-effort `CLAUDE_CLIENT_PRESENCE_FILE`
mtime), `lib/alerts.mjs` (pure `evaluateAlerts` + impure single-flight `checkAlerts`). One chokepoint
edit: `server.mjs` loads config at startup, adds a `cfg.alerts.enabled`-gated `setInterval`, and shares
a `costRollup()` helper between `/api/cost/rollup` and the budget alert.

- **Digest**: notifies when the `waiting-on-input` count changes (deduped); presence-suppressed.
- **Budget**: once per UTC day when the day's spend reaches `alerts.budget.thresholdUsd` (>0); not
  presence-suppressed. Alert state in `<config-dir>/alert-state.json` (never `~/.claude`).
- Built by a **worktree sub-agent → PR #9** (commit `a0b9ae7`), which also ran 2 adversarial reviews
  and caught a real read-modify-write race (slow cold scan let the interval queue overlapping ticks →
  fixed with a single-flight guard mirroring `inFlightScan`). I **code-reviewed the diff + independently
  re-verified** (dry-run boot on a throwaway port: both alerts fired exactly once; `:7433` untouched)
  before merging. **64/64 tests**, `node --check` clean. Merged as `06c2ce9`; tag `v1.2.0` + release.

To enable locally: write `~/.config/cc-orchestrator/config.json`, e.g.
`{"alerts":{"enabled":true,"checkIntervalMs":60000,"digest":{"enabled":true},"budget":{"thresholdUsd":50}}}`
and restart the server.
