## v1.6.0 — triage signals: blast radius + stuck (PR #28, Task #5)
`lib/health.mjs` derives, in its existing whole-transcript pass: `writes` + distinct
`filesTouched` (Edit/Write/MultiEdit/NotebookEdit), `destructiveBash` + `destructiveKinds`
(by category label only — never the raw command, which can carry a credential), and
`maxErrorStreak` (longest run of consecutive tool errors). Frontend: a red `⚠ N destructive`
badge, an amber `⟳ stuck` badge (streak ≥4), and a `✎ N files` health-line metric.

**The destructive detector was tuned against the real 109-session corpus** (the "run it
locally" discipline earned its keep): generic recursive `rm` is far too common to alarm, so
only genuinely irreversible ops (`git reset --hard`, `git push --force`, `git clean -f`,
`drop`/`truncate`, `dd of=`, `mkfs`, `chmod -R 777`) and **catastrophic-target** rm
(root/home/`*`/sudo) are flagged. Matching is **per command-segment** (so `rm -rf dist && cd
..` isn't a false positive), and `rm` is matched as a command, not the `--rm` flag
(`docker run --rm`). Result: ~14% destructive, ~17% stuck — meaningful, not noise.

## v1.7.0 — keyboard-first navigation (PR #29, Task #5)
`j`/`k` move a selection ring between cards; `Enter` opens chat, `c` context, `s` send, `f`
fork, `t` terminal (local), `/` focuses the filter, `Esc` clears, `?` toggles a shortcuts
cheatsheet (also the `⌨` header button). `⌘`/`Ctrl`+`K` still opens the launcher. Guards:
single-key shortcuts are suppressed while typing in a field, while a dialog is open, and
while a button/link/select is focused (so `Enter` on a card control doesn't also fire
`openChat`); `?` never stacks help on another modal; **arrow keys are left to native
scrolling**; the ring survives the periodic refresh and clears if its session is filtered out.
