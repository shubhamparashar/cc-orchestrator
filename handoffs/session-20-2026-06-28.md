# cc-orchestrator — 2026-06-28 — session 20

**v1.11.1 — copy confirmation shows on top of the chat dialog.** User-reported phone bug,
follow-up to the v1.11.0 click-to-copy feature.

## Bug
The "copied" feedback was a global `toast()`, which renders in the bottom-corner `.jobs`
panel (`position:fixed; z-index:40`). But the chat panel is a modal `<dialog>` opened with
`showModal()`, which the browser paints in the **top layer** — above any `z-index`. So the
copy toast appeared *behind* the open chat dialog and was invisible, most obviously on a phone.

## Fix
Copy feedback now lives **on the tapped bubble**, inside the dialog (same top layer), so it's
always visible. `copyBubble` calls a new `flashBubble(el, label)` that sets `el.dataset.flash`
+ a `flash` class; CSS `.chatmsg .bubble.flash::after` renders a centered accent pill
(`content: attr(data-flash)`) for ~1.1s (`toastIn` animation). `.bubble` gains
`position:relative` to anchor it. Success → "copied ✓", failures → "no clipboard" / "copy
failed". The global `toast()` call is gone from this path. `el.textContent` is read *before*
flashing, and the pill is `::after` content (not in `textContent`), so re-copying stays exact.

Frontend-only; `node --test` 137/137 unchanged. Small fix — self-reviewed (no untrusted
input, `::after`/dataset only, no `innerHTML`). No real-browser verify agent (Claude Preview
MCP still offline); verified via throwaway `:7475` markers + byte-identical-to-disk.
