# cc-orchestrator — 2026-06-29 — session 23

**v1.14.0 — closest-context injection for new live sessions (P2b).** Completes the second of the
two features from the improved build prompt (P2a shipped new-session launch in session 22). When
you start a NEW live session with a first prompt, the dashboard finds the closest related session
and offers to prime the fresh session with its saved context — so it starts oriented, not cold.

## What shipped (P2b)
- **A "Prior context" `<select>`** in the new-session dialog (`newLiveDialog`), populated from the
  existing relatedness ranker (`GET /api/related?q=<prompt>`), debounced as the prompt is typed
  (and on dialog open with a prefilled prompt). Reuses the ranker — no new search.
- **Default = the nearest match that has a saved context file** (`renderNewLiveContext` picks the
  first match with a `contextPath`). It deliberately does NOT auto-pick a no-context "continue"
  match — that would silently drop the typed prompt. "Start clean (no prior context)" is always
  the first option.
- **Transparent + overridable.** The chosen source is always shown in the select, and a hint
  (`newLiveContextHint`) states exactly what will happen — "reads that session's saved context,
  then your prompt" (prime) vs "resumes that session … your prompt above is not sent" (continue).
  Nothing is injected silently.
- **On Start (`newLiveGo`):** prime match → `{ cwd, level, prompt, contextSessionId }`; no-context
  match → `{ sessionId, cwd, level }` (resume the closest, no prompt); clean → no context id.

## Server + security
- `/api/live/start` gains an optional `contextSessionId`. Priming is scoped to
  `resumeId === null && initialPrompt && contextSessionId`: validate `isSessionUuid`, resolve the
  path **server-side** via `readContext()` (path = `join(CONTEXTS_DIR, <id>.md)`), then
  `initialPrompt = contextualPrompt(initialPrompt, ctx.path)`. **The injected path is never a
  client string** — only a real saved context file for an existing session can be primed; a
  forged/missing id fails safe (`ctx?.path` falsy → no prime). The primed text still rides as a
  discrete argv item (macOS) / `shq()`-escaped (Linux).
- `contextualPrompt(prompt, contextPath)` (extracted in `lib/actions.mjs`, the P2a groundwork)
  prepends `Read <path> for prior context, then: ` only when a path is present.
- **Audit:** the START line records `primed=<contextSessionId>`; the **prompt text is never
  logged**.
- Sub-agents can't be a prime source — they always carry `contextPath: null` and the frontend
  also filters `kind !== 'subagent'`.
