## v1.4.0 — A8 status chips + A4 compaction tokens (PR #25)
- **A8 — global status chips.** New `lib/status.mjs` (`claudeStatus({home})`) does one
  tolerant read each of `~/.claude/{.last-update-result.json, config.json,
  mcp-needs-auth-cache.json}` (missing / malformed / wrong-shape → null, never a thrown
  request) → `GET /api/status` → a header chip row (`#ccstatus`) showing the CC version,
  approval mode, and "N MCPs need re-auth" (server names in the tooltip). Exposes only
  version / approval-mode / MCP **names** — never the per-server id/token — so it's served
  to the authenticated phone dashboard too (the contrast is `/api/diag`, loopback-gated
  because it carries raw error stacks).
- **A4 — compaction token suffix.** The "N× compact" badge gains a "· ~969k" suffix: the
  pre-compaction context size (`compactMetadata.preTokens`) at the most recent compaction,
  via `lib/health.mjs` `lastCompactPreTokens` + a new `fmtTokens()`. **Note:** the
  "N× compact" half was *already shipped* under the A2 health work — the session-13 scout
  was stale on that point; only the token suffix remained, which is what this adds.

## v1.5.0 — A5 attribution (PR #26)
- `lib/health.mjs` now derives three `{name:count}` maps in its existing whole-transcript
  pass: `subagentTypes` (from the `Agent` tool's `input.subagent_type`), `skills` (from
  top-level `attributionSkill`), `mcpServers` (from top-level `attributionMcpServer`).
  They ride on `s.health` in `/api/sessions` — **no new endpoint**.
- Frontend: a `📜` skills badge and `🔌` MCP badge per card (busiest-first, capped, full
  `name×count` breakdown in the tooltip via `chipSummary`/`attributionBadges`), and the
  `⛬ subagents` tooltip gains a by-type breakdown (e.g. `Explore×4`). Every
  transcript-derived name is `esc()`'d into both body and `title`.
- **Security:** object keys now come from transcript fields, so a `bump()` helper guards
  them. The guard rejects every `Object.prototype` member (`key in {}` — catches
  `__proto__`, `constructor`, `toString`, `hasOwnProperty`, `valueOf`, …) **plus**
  `'prototype'` explicitly, because `prototype` lives on functions, not `Object.prototype`,
  so `in {}` alone misses it. (The reviewer's first suggested one-liner was `key in {}`
  only; the new proto-guard test — which includes inherited-method names — caught the gap.)
