# cc-orchestrator — 2026-06-26 — session 11

A9 — sub-agent transcript indexing. Folds the nested sub-agent transcripts
(`projects/<projectDir>/<sessionId>/subagents/**/agent-*.jsonl`) that the scanner
never read into both the cost model and the Tier-1 search index, from one data walk.

## What shipped (branch `feat/a9-subagent-indexing`, PR to `master`)

### New `lib/subagents.mjs`
Parent-join + per-file `(size,mtime)` caches (mirrors cost.mjs), reusing cost.mjs's
accumulation rather than duplicating it:
- `subagentFilesFor` — recursive `readdir({recursive:true})` over `subagents/`, filters
  `agent-*.jsonl` (excludes the orchestration-only `journal.jsonl`), recurses into
  `subagents/workflows/wf_*/`. Parent-join is the path component **before** `/subagents/`,
  matched exactly (a sessionId that is a string prefix of another can't collide).
- `readMeta` — tolerant (`{agentType:'subagent',description:'',toolUseId:null}` on
  missing/garbage).
- `subagentUsageByModel` / `subagentDateModel` — merged `{model:tokens}` and per-day×model
  maps summed across a session's sub-agent transcripts.
- `subagentDocsFor` — search docs `{id,parentSessionId,agentType,description,repo,body,model}`
  (dialogue-only tail, ~4KB cap).
- `subagentCountFor`, `pruneSubagentCaches`.

### `lib/cost.mjs`
Exported internals for reuse (no logic duplicated): `accumulateLine`, `consumeLines`,
new `accumulateDatedLine` (factored out of `usageByDateModel`'s loop — identical behavior),
new `mergeUsageByModel` (in-place token-add merge).

### `server.mjs`
- `attachCost` merges `subagentUsageByModel` into each session's own usage **before pricing**;
  merge is into a fresh map (`mergeUsageByModel(mergeUsageByModel({},own),sub)`) so neither
  cache is mutated. Adds `s.subagents` (count) + `s.subagentCost` to `/api/sessions`.
- `costRollup` feeds both own + sub-agent day×model maps to `rollupFromDaily` → cost-over-time
  modal AND budget alert include sub-agent spend.
- `relatedSessions` adds sub-agent docs as distinct hits (`kind:'subagent'`, unique `sub:<id>`
  doc id, navigates to parent), de-duped to one hit per (kind, parent).

### `lib/sessionIndex.mjs`
Appends `subagentDocsFor(...)` per session as `kind:'subagent'` entries
(description→title weight, agentType→tags, body→body — reuses rank.mjs weighting).

### `public/index.html`
Per-card "⛬ N subagents" chip (purple) when >0; sub-agent search hits render as
"↳ <agentType>: <description>" with a "View parent" action.

### Tests
`test/subagents.test.mjs` (16 tests) + fixtures under `test/fixtures/subagents-projects/`
(workflow nesting, missing/garbage meta, synthetic-line + toolUseResult-rollup exclusion,
prefix-collision sibling, additive merge). **87/87 green** (71 prior + 16 new).

## Cost-correctness (no double-count) — proven on real data
The parent transcript's `Agent` rollup is a model-less block on `type:user` lines;
cost.mjs only sums `type:assistant` + real model + usage, so it never counted it.
Verified on session `89385871-…`: 134 sampled sub-agent assistant `message.id`s have
**zero** overlap with the parent's 275 assistant-with-usage ids. Live numbers:
parent-only **$552.20** + sub-agents **$162.65** = `/api/sessions` cost **$714.86** (exact).
Fleet: 31/100 sessions have sub-agents; **~$1,080** previously-unattributed spend recovered
(`/api/cost/rollup` rose $25,718 → $26,799).

## Search payoff
Index went 100 → **654 entries** (554 sub-agent docs, ~5.5× the session count).
`/api/related?q=searchFromCollectionQuery` returns the sub-agent hit
"general-purpose: Trace searchFromCollectionQuery callers" attributed to parent `89385871`.

## Gotcha discovered (env, not a code bug)
`~/.claude/contexts/index.json` is a **single shared file**. While I tested on a throwaway
port, the live `:7433` LaunchAgent (old code) ran its own 5-min reindex and overwrote my
new-format index with the old 100-session one — so a query a few minutes after boot showed
0 sub-agent hits until I forced `/api/reindex`. Not a correctness issue (each server rebuilds
correctly), but two cc-orchestrator servers sharing one index file will clobber each other.
Worth considering a per-port index path or a single-writer guard later.
