## v1.10.0 — "Fork ⇢ auto" (skip-permissions autonomous fork)
Follow-up to v1.9.0: forking gave a live interactive session, but it still stopped for every
permission prompt. This adds a way to fork into a session that runs **autonomously**.

- `attachInTerminal`/`attachCommand` (`lib/actions.mjs`) gain a `skipPermissions` flag that
  appends the static literal `--dangerously-skip-permissions`; `/api/attach` (`server.mjs`)
  threads it through `Boolean`-coerced.
- A second local-only **"Fork ⇢ auto"** card button calls `attachAuto(id)`, which
  `confirm()`s first (a skip-permissions agent edits files / runs commands unattended) then
  `attach(id, null, true, true)` — fork + skip. `!IS_REMOTE`-gated like every terminal-launch
  action, so it cannot be triggered from the phone.
- User chose **full bypass** (not `acceptEdits`) and a **second button** (not a dialog
  checkbox). Headless `-p` forks are single-turn, so auto only applies to the terminal fork.
- New `test/actions.test.mjs` cases (skipPermissions alone; fork+skip flag order). 137/137.
- **code-reviewer: APPROVE** — `skipPermissions` is the only path that sets the flag, it
  appends a static literal only (no arbitrary-flag injection), defaults false everywhere,
  `sessionId` still UUID-validated + `shq()`-escaped, gating + confirm correct.

## v1.11.0 — click a chat message to copy it
- In the View chat panel, each `.bubble` is now click-to-copy. `copyBubble(e, el)` reads
  `el.textContent` (the original text — `esc()` entities decode on render, so no separate raw
  copy is stored), guards on `window.getSelection()` so drag-to-select still works, and toasts
  on success/failure (clipboard-unavailable handled). `cursor:pointer` + `title="click to
  copy"` + a subtle hover affordance.
- Frontend-only; `node --test` 137/137 unchanged. Small, low-risk change — self-reviewed
  (uses `textContent`, not `innerHTML`; no untrusted input; no backend surface) rather than a
  full review agent.
