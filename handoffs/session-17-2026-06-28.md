## Bug 1 — status filter didn't collapse the board
Picking a status in the `#statusFilter` `<select>` (e.g. `running`) filtered the cards but
**still rendered all four bucket columns and their `.colhead` headers**, the non-matching
ones showing "none". Root cause: `render()` filtered `shown` by status but then always
mapped over the full 4-entry `BOARD`.

**Fix:** when a status filter is active (`sf` truthy), `render()` sets `grid.className=''`
(dropping the `board` class so it falls back to the base responsive `#grid`,
`grid-template-columns: repeat(auto-fill, minmax(340px,1fr))`) and renders just
`shown.map(cardHtml)` — no `.col`/`.colhead` chrome, only the filtered sessions. The
empty-state and normal-board paths each now set their own `grid.className` (the single
top-of-function assignment was removed), and `applySelection()` runs in the filtered path
too, so the `j`/`k` selection ring keeps working. Clearing the filter restores the board.

## Bug 2 — "Open on your phone" dialog couldn't scroll or close
Opening the 📱 dialog and expanding the "First time? Set up Tailscale" `<details>`
(`#phoneGuide`) grew the dialog past the viewport. The dialog had `overflow:hidden` and
**no max-height**, so the overflow (including the Close button) was clipped — and on a
phone there's no Esc key, so there was genuinely no way to close it.

**Fix (applies to every dialog, not just the phone panel):**
- `dialog { max-height: 92vh; }` (kept `overflow:hidden` for the rounded-corner clip).
- `dialog[open] { display:flex; flex-direction:column; }` — **scoped to `[open]`** so a
  closed `<dialog>` keeps the UA `display:none`. A bare `dialog{display:flex}` would
  override that and make every dialog permanently visible; this was the one real risk and
  is why the rule is `[open]`-scoped.
- `dialog > .dlgpad { overflow-y:auto; min-height:0; }` — the padded body scrolls within
  the 92vh cap, keeping the footer/Close reachable.
- New global handler: clicking a dialog's backdrop (`e.target === d`) calls `closeDlg(d.id)`
  — the natural dismiss gesture on a phone. `e.target === d` is the canonical test: clicks
  on dialog *content* bubble up with `e.target` set to the child, so only true backdrop
  clicks close. (No `<select>` lives inside any dialog, so the native-dropdown footgun
  doesn't apply.)
