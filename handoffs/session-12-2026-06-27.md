## What PR #20 fixed (now released as v1.3.5)
- **CSRF on loopback** (`server.mjs`) — the `X-CC` first-party header is now required
  on every non-`GET`/`HEAD` request, not just authenticated *remote* POSTs. Closes a
  drive-by where a page the user visits issues a simple cross-origin POST to
  `127.0.0.1:7433` → `/api/send` (spawns `claude`) / `/api/attach` (opens a terminal).
- **`sessionId` UUID validation** (`server.mjs`) — `/api/send` + `/api/attach` reject a
  malformed `sessionId` with `400` before the spawn/attach path.
- **Prototype pollution** (`lib/cost.mjs`) — the attacker-controllable transcript
  `model` field, used as an object key during cost accumulation, can no longer be
  `__proto__`/`constructor`/`prototype`; those keys are skipped on every accumulation
  path (real model ids are never these).
- **Auth identity hardening** (`lib/auth.mjs`) — rate-limit key trusts `X-Forwarded-For`
  only from a loopback socket peer (local reverse proxy / Tailscale serve), so a direct
  peer can't rotate `XFF` to dodge the failed-auth limit; cookie parsing tolerates a
  malformed `%`-escape instead of throwing in the pre-handler path.
