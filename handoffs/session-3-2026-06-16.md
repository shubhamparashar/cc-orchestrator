## What this session did (the part NOT yet in commits/docs)

Reworked the F4 indexing from "thin index over the 3 sessions that had a context.md" into a **two-tier system**, and fixed a hard outage in context generation. All of this is **uncommitted** (see Git state).

### Tier 1 — free transcript-derived index over ALL sessions (DONE, live)
- New `lib/sessionIndex.mjs`: `buildSessionIndex()` walks every transcript, builds a rich index entry per session — `{sessionId, repo, cwd, gitBranch, title, tags[], goal, body, model, isWorktree, hasContext, contextPath, updated, sizeBytes}`. `body` = the user's own prompts (most-recent-first, capped 4 KB), cached on (size,mtime). **No model calls.** Writes `~/.claude/contexts/index.json`. Indexes 71 sessions in ~0.26 s.
- `lib/rank.mjs` rewritten to **BM25F-lite**: IDF + tf saturation (k1=1.2) + body length-norm (b=0.75), field boosts title6/tags5/repo4/goal3/body1.2, prefix-credit, recency + same-repo multipliers. Verified good: single-term queries (signoz, tailscale, vendor handpick, migration) are precise.
- `lib/contextStore.mjs`: removed old thin `rebuildIndex`; added `listContextSessions()` (set of session ids with a context.md) and `writeIndexFile(entries)`.
- `server.mjs`: `getSessions()` now derives `hasContext` from `listContextSessions()` (was wrongly true for all once the index held all sessions — bug fixed, verified hasContext=3 of 71); `relatedSessions()` ranks the rich index (+ folds in brand-new sessions); index auto-builds on startup + every 5 min + `POST /api/reindex` (single-flight `reindex()`). 
- `hooks/ctx-prompt.mjs`: uses new index shape (`body`, `contextPath` may be null).
- `jobs/rebuild-index.mjs` + `jobs/ctx-generate.mjs`: call `buildSessionIndex()` instead of the removed `rebuildIndex`.

### Context generation outage — fixed
- **Root cause:** `jobs/ctx-generate.mjs` hardcoded `--model claude-fable-5`, and **Fable 5 is currently access-gated ("currently unavailable")** → every `claude -p` exited 1 instantly. (The "unavailable" notice prints to stdout; old code only logged stderr, so it looked like an empty error.)
- **Fix (uncommitted):** default model → `claude-haiku-4-5` (cheapest available; `CC_CTX_MODEL` overrides); **fallback to the CLI default model** if the chosen model fails (so a single model going unavailable can't silently break everything again); dropped the now-pointless `--effort`; error capture now includes stdout. Verified one real generation (valid context.md, 15 s).

### In-flight RIGHT NOW
- **Context backfill is running** in the background: `node jobs/backfill-contexts.mjs --limit 100` (63 sessions, ~15 s each, haiku, sequential). Progress as of handoff: ~2 ok / 1 fail / 6 context.md files total. Log: `/tmp/cc-orch-ctx.log`. It's resilient (failures logged, re-runnable). **Check on it:** `grep -c ' ok ' /tmp/cc-orch-ctx.log`; `pgrep -f backfill-contexts`. If it died, just re-run the same command (skips sessions that already have a context.md). The `~3 fail` so far are mostly "not enough dialogue to summarize" (tiny sessions) — expected, not a bug.
